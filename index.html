Autonomous Navigation Robot +

A fully autonomous robot capable of mapping unknown environments and navigating through complex obstacles using advanced SLAM algorithms and multi-sensor fusion.

ROS Computer Vision SLAM Sensor Fusion LiDAR

Project Overview

This project addresses the challenge of autonomous navigation in unstructured and dynamic environments. The robot is designed to create real-time maps of its surroundings, localize itself within that map, and plan efficient paths while avoiding obstacles. As the lead developer, I was responsible for the system architecture, sensor integration, and navigation algorithms.

Hardware Components

  • Base Platform: Custom 4-wheel differential drive chassis with high-torque DC motors
  • Sensors: 360° LiDAR (RPLiDAR A2), RGB-D Camera (Intel RealSense D435), 9-DOF IMU, Wheel encoders
  • Compute: NVIDIA Jetson Xavier NX for real-time processing
  • Power System: 12V LiPo battery pack with 10000mAh capacity and custom power distribution board
  • Custom Components: 3D printed sensor mounts and protective housing designed for optimal sensor placement and environmental protection

Electronics

Core Systems:

  • Custom PCB for motor control and sensor interfacing
  • STM32F4 microcontroller for low-level control
  • Voltage regulation system for multiple power rails (5V, 3.3V)
  • CAN bus communication between subsystems

Circuit Design:

  • H-bridge motor drivers with current sensing
  • Signal conditioning for analog sensors
  • Safety cutoff system for emergency stops
  • Battery management system with charge monitoring

Electronics block diagram showing system interconnections

Software Architecture

The software system is built on ROS (Robot Operating System) with a modular architecture divided into perception, localization, planning, and control subsystems:

ROS Noetic C++ Python OpenCV PCL TensorFlow
  • Perception: Sensor data processing, obstacle detection, environment feature extraction
  • Localization: Adaptive Monte Carlo Localization (AMCL) with sensor fusion from LiDAR, camera, IMU and odometry
  • Mapping: Gmapping for 2D grid maps, RTAB-Map for 3D point cloud mapping
  • Path Planning: Global planning using A* algorithm, local planning using Dynamic Window Approach
  • Control: PID controllers for velocity and position control with dynamic parameter tuning

Software architecture diagram showing major nodes and data flow

Development Process

The development followed an iterative approach with several prototyping phases:

  • Phase 1: Platform design and basic control system implementation
  • Phase 2: Sensor integration and data acquisition pipeline development
  • Phase 3: SLAM implementation and testing in controlled environments
  • Phase 4: Path planning and obstacle avoidance algorithm development
  • Phase 5: System integration, optimization, and extensive testing in various environments

Key challenges included sensor calibration for optimal fusion, managing computational constraints on the embedded platform, and robust operation under varying lighting and terrain conditions.

6-DOF Robotic Manipulator +

A precision 6-degree-of-freedom robotic arm with vision-based object recognition and advanced kinematics for automated pick-and-place operations in manufacturing environments.

Inverse Kinematics Computer Vision Servo Control Path Planning

Project Overview

This project focuses on the development of a 6-DOF robotic manipulator designed for precision pick-and-place operations. The system combines computer vision for object recognition with advanced kinematics algorithms to achieve accurate positioning. My role involved designing the mechanical structure, implementing the control system, and developing the vision-based object detection.

Hardware Components

  • Structure: Custom-designed aluminum frame with 3D printed joints and end effector
  • Actuators: 6 high-torque servo motors (MG996R for base joints, DS3218 for wrist)
  • End Effector: Pneumatic gripper with pressure sensors
  • Sensors: RGB Camera (Logitech C920), force/torque sensor at the wrist, joint position encoders
  • Control Board: Arduino Mega for low-level control, Raspberry Pi 4 for high-level processing

CAD model of the robotic arm showing joint configuration

Electronics

Power System:

  • 5V/10A and 12V/5A power supply units
  • Servo power distribution board with noise filtering
  • Emergency stop circuit with hardware interlocks

Control Electronics:

  • PCA9685 16-channel PWM controller for servo coordination
  • Custom signal conditioning for analog feedback
  • I2C communication between microcontrollers
